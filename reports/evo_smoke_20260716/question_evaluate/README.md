# 객관식 Solver 평가 테스트 결과

## 한눈에 보는 결론

이번 테스트에서는 영상 4개로 객관식 문제 4개를 만들고, Solver가 각 문제를 10번씩 풀게 했다.

- 영상 입력 성공: 4개 중 4개
- 생성한 Solver 답변: 총 40개
- 사람이 정한 기대 정답: `B, B, C, B`
- Solver 다수결 정답: `B, B, C, B`
- 기대 정답과 다수결 정답이 일치한 문제: 4개 중 4개
- 학습 데이터로 남은 문제: 4개 중 4개
- Hugging Face 업로드: 실행하지 않음

즉, 네 문제 모두 전체 영상을 Solver에 전달하고 답변 10개를 생성하는 데 성공했다. 네 문제의
다수결 정답도 기대 정답과 모두 같았다.

## 테스트가 진행된 순서

1. Questioner가 영상에서 연속된 8프레임만 보고 문제를 만든다고 가정한다.
2. Solver에는 같은 영상의 전체 16프레임과 A/B/C/D 문제를 전달한다.
3. Solver가 문제 하나당 답변을 10개 생성한다.
4. 10개 답변에서 가장 많이 나온 답을 다수결 정답으로 정한다.
5. 다수결 비율이 0.3 이상 0.8 이하인 문제만 학습 데이터로 남긴다.
6. 영상 경로와 Questioner가 봤던 8프레임의 시간 구간을 함께 저장한다.

## 점수를 읽는 방법

여기서 `score`는 문제의 정답률이 아니다. Solver가 10번 답했을 때 다수결 정답과 같은 답이
몇 번 나왔는지를 나타내는 일치 비율이다.

예를 들어 다음 결과는 Solver가 10번 중 4번 B를 답했다는 뜻이다.

```text
다수결 정답: B
같은 답이 나온 횟수: 4번
score: 4 / 10 = 0.4
```

논문 방식에서는 `0.3 <= score <= 0.8`인 문제를 남긴다. 0.3보다 낮으면 Solver의 답이 너무
불안정한 문제이고, 0.8보다 높으면 현재 Solver에게 너무 쉬운 문제로 본다.

## 문제별 결과

| 번호 | 문제 내용 | 기대 정답 | 다수결 정답 | 같은 답 횟수 | score | Questioner가 본 시간 구간 |
|---:|---|:---:|:---:|---:|---:|---:|
| 1 | Red-Black Tree에 빨간 간선이 나타나는 이유 | B | B | 4/10 | 0.4 | 4.67~11.00초 |
| 2 | 위 점액층의 주요 기능 | B | B | 4/10 | 0.4 | 2.00~8.96초 |
| 3 | 마지막 장면에서 강조되는 트리 노드 값 | C | C | 6/10 | 0.6 | 10.63~19.97초 |
| 4 | 식도와 위가 연결되는 지점의 구조 | B | B | 5/10 | 0.5 | 7.96~14.96초 |

네 문제 모두 score가 0.3 이상 0.8 이하이므로 학습 데이터로 남았다.

## 잘못된 `<segment>` 답변 처리

Solver는 정답과 관련 시간 구간을 각각 다음 형식으로 출력해야 한다.

```text
정답: \boxed{B}
관련 구간: <segment>2.0s-5.0s</segment>
```

하지만 원본 답변 40개 중 7개는 시간 구간을 정답 칸에도 넣었다.

```text
\boxed{<segment>2.0s-5.0s</segment>}
```

현재 `evaluate.py`는 `\boxed{}` 안에 `<segment>` 태그가 있으면 해당 답변 후보를 다수결에서
제외한다. 필터 적용 후 후보 정답에 `<segment>` 태그가 남은 경우는 0개다.

이번 요청에 따라 태그 없이 시간만 쓴 다음 형태는 아직 제외하지 않았다.

```text
\boxed{2.0s-5.0s}
```

이러한 답변은 2개 있었지만 어느 문제에서도 다수결 정답이 되지는 않았다.

## upload 테스트 결과

`upload.py`를 사용해 실제 업로드 직전의 로컬 학습 파일까지 만들었다.

- 평가 결과 입력: 4개
- score 범위를 통과한 문제: 4개
- 최종 학습 데이터: 4개
- 잘못된 메타데이터로 거부된 문제: 0개
- Hugging Face에 실제 업로드: 하지 않음

최종 학습 데이터에는 다음 정보가 유지됐다.

- 문제와 다수결 정답
- 원본 영상 경로
- 전처리된 16프레임 `.pt` 경로
- Questioner가 문제를 만들 때 본 8프레임의 시작·종료 시간
- 문제 종류와 confidence score

학습에 필요하지 않은 10개 후보 답변과 긴 원문 Solver 응답은 최종 학습 파일에서 제거됐다.

## 파일을 보는 순서

1. `README.md`: 사람이 읽기 위한 전체 설명
2. `mc4_summary.json`: 주요 숫자를 모은 요약
3. `mc4_train.json`: upload 테스트로 만든 최종 학습 데이터
4. `mc4_results.json`: 40개 후보와 원문 응답을 포함한 상세 결과
5. `mc4_input.json`: A/B/C/D로 수정한 입력 문제 4개
6. `mc4_train_rejected.json`: upload 검사에서 거부된 데이터 목록이며 현재는 비어 있음

## 아직 하지 않은 부분

이번 `evaluate` 단계에서는 IoU를 계산하지 않았다.

- `target_segment`: Questioner가 문제를 만들 때 본 실제 시간 구간
- `candidate_segments`: Solver가 답변마다 예측한 관련 시간 구간

두 값은 결과 파일에 보존돼 있다. 다음 Solver GRPO 학습 단계에서 두 구간의 IoU를 계산해 reward에
추가하게 된다.
